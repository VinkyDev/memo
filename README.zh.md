# memo

> **持久化工作记忆，陪伴长时间的 Agent 会话。**
> 每个项目只需一个轻量的 `MEMORY.md`，在上下文压缩后依然留存——不再重复燃烧 Token 去重新发现你已经知道的东西。

## 问题

模型的上下文有限。在任何非 trivial 的功能开发中，都会发生这样的事：

1. 你花了半个上下文窗口帮模型*理解*代码库。
2. 对话越来越长。压缩触发。来之不易的理解烟消云散。
3. 下一轮又燃烧 Token，重新发现同样的文件、同样的约束、同样的坑。

`CLAUDE.md` 和 `AGENTS.md` 解决的是另一个问题——*静态*的项目约定（"我们这里怎么做事"）。它们无法回答"我刚才做到哪了、刚学到了什么、今天已经踩过两次的坑是什么"。

## 思路

`memo` 引入了第三个文件：**`MEMORY.md`** —— 鲜活的、任务范围的、极度精简的工作记忆。

| 文件 | 范围 | 生命周期 | 内容 |
|---|---|---|---|
| `CLAUDE.md` / `AGENTS.md` | 项目级规则 | 永久 | "我们这里怎么做事" |
| `README.md` | 人类入职 | 永久 | 项目是什么 |
| **`MEMORY.md`** | **当前项目中的活跃工作** | **工作进行中** | **目标 · 核心上下文 · 进度 · 决策 · 陷阱** |

Agent 会在整个会话中自动维护它——当你完成里程碑、踩到坑、或做出值得记住的决策时。在新会话中，`/memo` 会在几秒内把它读回来，你就能准确接上上次离开的地方。

> **一条规则**：每一行节省的 Token 必须比它本身消耗的更多。30 行纯信号的备忘录胜过 300 行流水账。

## 安装

### 通过 `skills.sh`

```bash
npx skills add VinkyDev/memo
```

skill 会被复制到 `~/.claude/skills/memo/`，在任何 Agent 会话中可用。

### 手动安装

将 `SKILL.md` 放入你的 Agent 的 skills 目录（或它读取自定义 skill 定义的任何位置）。

## 用法

只有一个命令，可选地带上后续提示：

```text
/memo
/memo <你想做的下一件事>
```

这就是整个界面。skill 会自动处理其余部分：

- **第一次在项目中** → 用一句话询问目标，写入 `MEMORY.md`，然后处理你的提示。
- **`MEMORY.md` 已存在** → 加载它（静默——不输出文件内容），确认当前光标位置，然后处理你的提示。
- **工作推进中** → 当里程碑、决策或值得记住的陷阱出现时，skill 会自动更新 `MEMORY.md`。你无需主动要求。

---

## 与 /clear 和 /compact 配合使用

`MEMORY.md` 是让上下文重置变得廉价的安全网。一旦建立：

- 放心使用 `/clear` 或 `/compact`——关键上下文保存在文件里，不在对话中。
- 更短的上下文 = 更快的响应、更低的成本、更少的压缩意外。
- memo 模式鼓励积极的上下文卫生：提早压缩、频繁压缩。

---

## 如何保持精简

`MEMORY.md` 在设计上就被强制保持**小巧**：

- ~300 行软上限；~400 行硬上限（超过时 Agent 会先裁剪再添加）
- 每部分上限（核心上下文 ≤ 8 条，决策 ≤ 5 条）
- skill 内置严格的包含/排除规则——不放代码 dump、不放聊天记录、不重述项目约定

如果你要求 Agent 把 200 行的堆栈跟踪塞进记忆，它会拒绝。这就是它的意义。

## 什么该放在哪里

| 问题 | `CLAUDE.md` / `AGENTS.md` | `MEMORY.md` |
|---|---|---|
| "我们使用 TypeScript strict 模式" | ✅ 静态约定 | ❌ 已经永久已知 |
| "Auth 在 `src/auth/`" | ✅ 项目地图 | ❌ 稳定信息 |
| "RP ID `localhost` 只在 `:3000` 上有效 —— `127.0.0.1` 会静默失败" | ❌ 任务特定的坑 | ✅ 陷阱，能为未来会话节省 30 分钟 |
| "我们选择 `@simplewebauthn/server` 而不是 `fido2-lib`，因为…" | ❌ 决策会淹没在聊天中 | ✅ 决策 + 理由 |
| "`register` 路由做了一半" | ❌ 不是项目规则 | ✅ 进度光标 |

**`CLAUDE.md` 是项目的 README。`MEMORY.md` 是进行中的工作的 README。**
